# CLAUDE.md — Churn Rate Analytics (Scenario 04)

## Project in one sentence
A SaaS churn rate analytics system: synthetic dataset → calculation engine → Streamlit dashboard → NL eval harness, all versioned on a single canonical metric definition.

## How to run things

```bash
# install
pip install -r scenario-04/requirements.txt

# regenerate data
python scenario-04/generate_data.py

# unit tests
pytest scenario-04/engine/tests/ -v

# dashboard (opens browser at localhost:8501)
streamlit run scenario-04/dashboard/app.py

# eval harness
python -m eval.runner              # needs ANTHROPIC_API_KEY or Claude Code auth
python -m eval.runner --dry-run    # print questions only
python scenario-04/eval/mock_runner.py  # fully offline

# presentations
python scenario-04/generate_ppt.py
python scenario-04/generate_html.py
```

## Core conventions

### Never touch these without updating DEFINITION_VERSION
- `engine/definition.py` — all numeric thresholds live here
- If you change a threshold, bump `DEFINITION_VERSION` from `"1.0"` to `"1.1"` etc.
- The dashboard footer, glossary, CSV export, and every `ChurnResult` will update automatically.

### calculator.py is pure — keep it that way
- `calculate_churn_rate()` takes DataFrames, returns `ChurnResult`. No file I/O, no HTTP calls.
- All side effects (loading CSVs, caching) belong in `data_loader.py` or `metrics.py`.

### Charts always use black labels
- All Plotly figures use `color="#111111"` for tick labels, axis titles, and bar text.
- Use `title=dict(text=..., font=dict(color=...))` — not the deprecated `titlefont=`.

### data/ contains synthetic data only
- Never commit real customer data. The CSVs are generated by `generate_data.py`.
- If connecting to a real database, add CSVs to `.gitignore` and update `data_loader.py`.
